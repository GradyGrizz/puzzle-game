# Decisions made with full authority — flagged for your review

Everything below shipped with my best judgment. None block TestFlight,
but each is a place where you might have chosen differently. One pass
through this list and you've reviewed every judgment call in the build.

## Product / design

1. **Name: "DELVE — The Sunken Keep."** Short, retro, App-Store
   searchable. Trivial to rebrand (one wordmark, title strings, icon).
2. **No forced ads, ever.** Revenue = coin IAPs + opt-in rewarded ads +
   coin-bought cosmetics. This is lower short-term revenue than
   interstitials but far better retention/reviews. See MONETIZATION.md.
3. **Challenge Mode fail state = move budget** (~1.6× the generator's
   own solution + slack). Alternatives considered: no fail state
   (leaderboard meaningless), deadlock-ends-run (brutal). The budget is
   generous but creates real tension. Tunable in `gen.js` (`budget`).
4. **Timed Rush = fixed 5-stage gauntlet** (depths 2,4,6,8,10) rather
   than an endless timer. Cleaner leaderboard semantics ("fastest run")
   and a natural session length (~3-6 min).
5. **Challenge/Timed puzzles use only blocks + switches** (no cracks,
   fire, etc.). Keeps the on-device generator verifiably solvable and
   fast. Story mode owns the exotic mechanics. Could be expanded later.
6. **The chest must be opened before the exit unseals** on relic levels
   — prevents a player skipping their sword and soft-locking chapter 2.
7. **Story = 31 levels** (8+6+6+6+7). A comfortable 2-4 hour campaign.
   More chapters (a "boots" relic was sketched: spikes/dash) are easy to
   add — the engine already supports unused item slots.
8. **Difficulty saturates around depth ~30** in Challenge (board size
   caps at 11×9, 4 blocks). Depth escalation past that is more of the
   same rather than strictly harder. Acceptable for a leaderboard game;
   bigger boards would need UI work for smaller tiles.
9. **Hints are story-only** (generated levels can exceed the live
   solver's budget) and consumable (3 for 60 coins) — a deliberate
   frustration-to-coin-sink conversion.
10. **Economy numbers** (5/clear, +10 first clear, 3/challenge depth,
    12/timed run, skins 150-400, themes 200): tuned so a free player
    owns their first skin in session one. Not playtested at scale —
    watch this in TestFlight feedback.

## Technical

11. **Plain JS, no bundler, no framework.** The whole game is ~5k lines
    of dependency-free code — trivially wrappable, auditable, and fast.
12. **All art except the hero sheet is procedural canvas drawing.**
    Guarantees palette cohesion and made themes nearly free. The hero
    sprite sheets (`hero2.png`, `push_left.png`) are the established
    assets and remain the only bitmaps.
13. **All audio is synthesized WebAudio chiptune** — zero audio files,
    zero licensing. The three music loops are short and deliberately
    sparse; a composer pass would elevate them (they're serviceable,
    not memorable).
14. **Leaderboards are local-device** with Game Center sync stubbed for
    the native build. No backend/accounts anywhere — biggest scope
    decision in the project. Global leaderboards would need a service.
15. **Skins recolor the sprite's green pixels at runtime** via canvas
    `getImageData`. Works under Capacitor/http; on `file://` dev it
    silently falls back to default colors.
16. **Saves live in `localStorage`** (versioned blob). The TestFlight
    doc flags mirroring to Capacitor Preferences as required — WKWebView
    can evict localStorage.
17. **Dark levels assume the lantern** (radius is constant). The lantern
    is guaranteed by progression gating, so a "no-lantern darkness"
    state is unreachable.
18. **App icon is generated by the game's own art code** (chest +
    wordmark). It reads as "retro dungeon" but a hand-drawn 1024 icon
    would market better — easy swap at `icons/icon-1024.png`.

## Known limitations (not bugs)

- Canvas UI is invisible to VoiceOver/screen readers (typical for
  canvas games; noted in TESTFLIGHT.md QA).
- Landscape works but is unoptimized; portrait is the designed
  experience (locked in the iOS wrapper).
- Replaying a completed story level skips its intro dialog by design;
  there is no dialog-log/replay feature.
- The challenge leaderboard records a quit mid-depth as the last fully
  cleared depth — quitting at depth 5 with 4 cleared records 4.
